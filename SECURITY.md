# Security Policy

## Reporting Security Issues

If you discover a security vulnerability, please email security@elpruscripts.dev with:
- Description of vulnerability
- Steps to reproduce
- Potential impact

Please do not publicly disclose the vulnerability until we've addressed it.

## Security Best Practices

- All scripts are verified and tested on PC
- Never share your scripts with untrusted sources
- Always use the official key system from work.ink
- Update scripts regularly

## Compliance

This site implements:
- Content Security Policy (CSP) to prevent XSS attacks
- X-Frame-Options to prevent clickjacking
- HTTPS enforcement
- Secure headers and permissions policy
